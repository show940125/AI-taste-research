# 第一輪 Prompt 模板包

這份模板包是給第一輪探索測試直接使用的。它把五組條件寫成可直接貼到 `OpenAI/Claude` 的版本，欄位和命名跟前面的研究框架一致。

## 使用方式

1. 從 [03-題組與評估/第一輪探索題組.md](../03-題組與評估/第一輪探索題組.md) 選一題。
2. 把該題的 `task_type`、`target_audience`、`length_target`、`content_constraints`、`tone_constraints`、`source_material` 填入對應欄位。
3. 同一題固定只換 prompt 條件，不改其他欄位。
4. 正式盲評前，刪掉輸出中的 `audit_notes`。

## 共通欄位

```text
task_type:
target_audience:
length_target:
content_constraints:
tone_constraints:
source_material:
```

## 1. Baseline

```text
你是一位中文寫作者。請根據以下材料完成任務。

任務類型：{task_type}
讀者：{target_audience}
字數：{length_target}
內容限制：{content_constraints}
語氣限制：{tone_constraints}

材料：
{source_material}

要求：
- 使用台灣中文
- 輸出一篇完整成品
- 最後另列 `audit_notes`，用短句寫出你自己看到的風格風險
```

## 2. Persona-only

```text
你現在是一位中文論說文作者，文風要求如下：

- 明白、節制、講理，有判斷
- 長短句交錯，不要整篇一樣齊
- 先交代表面合理性，再補條件、範圍、因果
- 優先用動詞與名詞，少堆抽象名詞
- 避免聊天腔、討好腔、口號式總結、過度工整的三段式

任務類型：{task_type}
讀者：{target_audience}
字數：{length_target}
內容限制：{content_constraints}
語氣限制：{tone_constraints}

材料：
{source_material}

輸出：
1. 正文
2. audit_notes
```

## 3. Few-shot-only

先固定用下面三個示例，不要每題換。

### Few-shot 示例 1

```text
題目：評析「遠端工作讓協作一定變差」這種說法。

示例輸出：
把問題講成「遠端一定讓協作變差」，其實太省事。差的往往不是距離，是流程。會議開得像點名，文件散在聊天視窗裡，責任邊界又模糊，人在同一間辦公室也一樣亂。

遠端工作確實拿掉了很多走到座位旁邊就能解決的小事。可它也逼團隊把含糊的地方寫清楚。該留紀錄的留紀錄，該指定責任的指定責任，反而把原本靠默契硬撐的問題照出來。

所以遠端不是萬靈丹，也不是原罪。它只是把管理能力攤開給人看。
```

### Few-shot 示例 2

```text
題目：把一段學術摘要改寫成一般讀者看得懂的說明。

示例輸出：
這篇研究講的事情不玄。研究者追蹤了兩組學生一整年，想看每天滑短影音的時間，跟專注力變化有沒有關係。結果有相關，但沒有一些新聞標題說得那麼乾脆。

問題不在「看短影音」四個字本身。差別更像出在使用情境。零碎時間看、看完就停，影響有限；睡前一路滑到停不下來，第二天的注意力就容易掉。

所以這篇研究真正補上的，不是道德判決，而是使用方式的細節。
```

### Few-shot 示例 3

```text
題目：把英文制度說明改寫成中文短評。

示例輸出：
這套制度表面上在講效率，骨子裡講的是誰能決定流程。文件把每一步都寫得很乾淨，像是權責分明，實際上卻把例外情況塞到後面的小條款裡。真出事時，還是要靠人臨場判斷。

這不是說制度沒用。問題在於它把衝突藏起來，讓人誤以為只要照表操課，事情就會自己變順。
```

## 4. Rewrite-pipeline

```text
請依三步完成任務。

Step 1. 先整理材料的核心意思與不能漏掉的資訊，不要照原句序重寫。
Step 2. 依中文習慣寫出初稿。先求清楚，再求節奏與作者感。
Step 3. 檢查初稿，逐條修正：
- 題面詞是否重用過多
- 連接詞是否過密
- 主語代詞是否補得太滿
- 是否有過長的「的」字鏈
- 是否殘留英文被動、抽象名詞與從句痕跡
- 是否有聊天腔、討好腔、模板收尾
- 是否整篇段落太齊、太像同一模子

任務類型：{task_type}
讀者：{target_audience}
字數：{length_target}
內容限制：{content_constraints}
語氣限制：{tone_constraints}

材料：
{source_material}

輸出：
1. 正文
2. audit_notes
```

## 5. Classical/Vernacular Pivot Pipeline

```text
請依四步完成任務。

Phase 1. 先讀懂材料，只抓意思與關係，不照原句序。
Phase 2. 用半文言或古典筆意把事情重述一遍。重點是打散英文句法、抽象名詞鏈與現成模板骨架，不求華麗，不求堆典故。
Phase 3. 只根據 Phase 2 的內容，重寫成新文化白話式中文。要求明白、短句、節制、有人的判斷，不要聊天腔，不要公文腔。
Phase 4. 做最後稽核，修掉殘留的翻譯腔、聊天腔、過整齊的段落節奏與空泛姿態詞。

任務類型：{task_type}
讀者：{target_audience}
字數：{length_target}
內容限制：{content_constraints}
語氣限制：{tone_constraints}

材料：
{source_material}

輸出：
1. 正文
2. audit_notes
```

## 6. Final Audit Pass

這一步用於五組條件的最後一輪清理，尤其適合 `rewrite-pipeline` 與 `pivot pipeline`。

```text
請只做最後稽核，不要重寫立場與內容重心。

逐條檢查：
1. 刪去空泛姿態詞與模板收尾
2. 打散過整齊的句法與段落
3. 去掉翻譯腔：代詞過滿、被動殘影、連接詞過密、「的」字鏈、抽象名詞
4. 去掉聊天腔與協作式尾巴
5. 若某處已自然，保持原樣，不要為了變化而硬改

輸出：
1. 修正版正文
2. audit_notes
```

## 建議執行順序

若第一輪只跑三組，建議先跑：

1. `baseline`
2. `rewrite-pipeline`
3. `classical/vernacular pivot pipeline`

若時間夠，再補：

4. `persona-only`
5. `few-shot-only`

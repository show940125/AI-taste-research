# 強力 pivot 版 anti-AI 味 prompt 模板

這版不是日常預設，而是重手工具。它把 `classical/vernacular pivot pipeline` 直接拉成主幹，專門處理英譯中殘影很重、抽象說理一路沿英文骨架直推、一般 `rewrite` 已經壓不下去的文本。它的優點是能換骨，缺點是容易讓篇幅縮短、語勢變緊，所以不該亂用。

## 適用情境

- 英譯中重寫
- 抽象說理題
- 讀起來明顯像英文先想好再翻成中文
- 初稿雖然通順，骨架卻很像模型或翻譯稿

## 主模板

```text
你現在要做的不是順稿，而是換掉中文背後的骨架。

任務：
請依下面材料寫出一篇 {task_type}，面向 {target_audience}，長度約 {length_target}。

必守內容：
{content_constraints}

語氣要求：
{tone_constraints}

材料：
{source_material}

請按這個流程處理，但最後只輸出成品與 audit_notes，不要把中間推理攤開。

Phase 1｜脫離原句序
- 先只抓事情、關係、判斷與不可遺漏的資訊。
- 不保留原文句序，不照原文從句結構重寫。
- 先辨認哪些地方有被動句、抽象名詞鏈、代詞過滿、顯性連接過密。

Phase 2｜中間轉寫
- 先在心裡用極簡、半文言或古典筆意把內容重述一遍。
- 目的只有一個：切斷英文骨架與現成模板骨架。
- 不求古雅，不堆典故，不模仿古人，不輸出這一步。

Phase 3｜轉回現代白話中文
- 只根據 Phase 2 整理出的意思，重寫成現代中文。
- 使用台灣中文。
- 句子要短中有長，讓節奏自然變化。
- 優先用動詞與名詞，不要讓抽象名詞一路疊上去。
- 能省略主語就省略。
- 少用「因此」「然而」「換言之」這類顯眼連接詞。
- 避免過整齊的對照句、排比句與口號式收尾。
- 先把事情說實，再補條件、範圍與判斷。

Phase 4｜最後去味
- 刪去聊天腔、助理腔、協作式尾巴。
- 檢查是否還留有被動殘影、長「的」字鏈、題面詞重用、抽象名詞鏈。
- 若某段太像模型慣走的平滑說理，改寫那一段的呼吸與重心。
- 若語勢過猛、篇幅縮太多，適度補回必要資訊。

輸出格式：
## 正文
（直接給成品）

## audit_notes
- 用 3 到 6 條短句列出你實際切掉了哪些英文骨架或模板痕跡。
```

## 這版真正處理的是什麼

- 英語句法直接投影到中文
- 抽象名詞鏈與被動殘影
- 說理一路沿概念直推的平滑模板
- 中文表面通順，骨架卻不是中文的問題

## 風險

- 容易把篇幅壓短
- 容易讓語勢變硬
- 若原材料其實已是自然中文，硬走這版反而會改出加工痕跡

## 使用提醒

這版要在真的有英文骨架時再出手。若你只是覺得文字有點太像助理，先用融合穩健版。強力 pivot 的價值在於換骨，不在於把每篇文章都寫得比較有勁。

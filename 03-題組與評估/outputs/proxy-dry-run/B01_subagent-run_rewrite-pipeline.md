# 第一輪正式結果（subagent-run）

這份結果納入第一輪正式測試紀錄。writer 來源是 subagent，目的在於建立可比較的正式首輪樣本；它不是 OpenAI 或 Claude API 直出結果。

## 正文

研究者追蹤 624 名上班族將近一年，想看一件很多人都有感的事：通知一直跳，工作是不是更容易被切碎。做法很直接。受試者在自己的裝置上裝記錄工具，每週再填問卷，讓研究團隊對照通知頻率、切換任務的次數，以及他們主觀感受到的掌控感。

結果看起來不意外。通知越頻繁，受試者越常在不同工作之間跳來跳去，也越容易覺得節奏不受自己控制。但研究沒有把責任全推給裝置。若團隊平常就把回覆規則講清楚，哪些事要立刻處理，哪些可以晚一點回，通知帶來的干擾會明顯小一些。

這讓問題的重心換了位置。干擾當然和工具設計有關，可真正把人拖垮的，常是工作場所默認的即時回覆壓力。研究限制也要記著：它看見的是關聯，不是單一原因，而且掌控感仍有主觀成分。

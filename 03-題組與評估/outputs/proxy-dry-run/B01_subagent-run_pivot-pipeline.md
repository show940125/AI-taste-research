# 第一輪正式結果（subagent-run）

這份結果納入第一輪正式測試紀錄。writer 來源是 subagent，目的在於建立可比較的正式首輪樣本；它不是 OpenAI 或 Claude API 直出結果。

## 正文

這篇研究看的，是辦公室裡一種很細碎、卻很磨人的消耗。624 名上班族被追了 11 個月。研究團隊讓他們在裝置上裝記錄工具，再配每週問卷，想知道通知一多，人的注意力是不是就更容易碎掉。

答案大致是會。通知頻繁的人，工作切來切去的次數較多，也更常覺得自己抓不住節奏。可研究沒有停在這裡。若一個團隊平時就把回覆規矩講明白，哪些訊息要快，哪些可以慢，通知的干擾會收斂不少。

這意思很簡單。問題不全在手機，也不全在個人自制。很多時候，是工作場所把「隨時要回」當成默契，才把每一次提醒都變成壓力。這篇研究沒有說通知一定害人，它只是把責任分得更準一些。

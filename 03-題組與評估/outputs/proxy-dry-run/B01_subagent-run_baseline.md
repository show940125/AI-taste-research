# 第一輪正式結果（subagent-run）

這份結果納入第一輪正式測試紀錄。writer 來源是 subagent，目的在於建立可比較的正式首輪樣本；它不是 OpenAI 或 Claude API 直出結果。

## 正文

這篇研究追蹤了 624 名上班族，時間長達 11 個月，想看裝置通知是不是會讓人的注意力更碎。研究做法不複雜：受試者在自己的裝置上安裝記錄工具，研究團隊再配合每週問卷，觀察通知頻率、任務切換和工作掌控感之間的關聯。

結果顯示，通知越多的人，通常越常被打斷，也越容易覺得自己沒那麼能控制工作節奏。這個方向很符合很多人的直覺。不過研究還補了一個重要條件：如果團隊對回覆時機有明確規範，像是哪些訊息要立刻回、哪些可以晚一點處理，通知造成的干擾就會減弱。

換句話說，問題不只在手機或電腦跳出多少提醒，也跟工作場所怎麼期待回覆有關。這篇研究的價值，在於它把注意力分散這件事從單純的個人習慣，拉回到工作規則與團隊文化一起看。

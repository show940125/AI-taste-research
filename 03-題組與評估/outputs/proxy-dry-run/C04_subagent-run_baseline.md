# 第一輪正式結果（subagent-run）

這份結果納入第一輪正式測試紀錄。writer 來源是 subagent，目的在於建立可比較的正式首輪樣本；它不是 OpenAI 或 Claude API 直出結果。

## 正文

自動化不只是移除勞動，更常見的情況是重新分配勞動。有些任務確實會消失，但另一些工作則會被往下游推，交給使用者、外包者，或者那些仍然留在系統中的工作者。由於這些新增加的勞動往往零碎、分散，而且名字不明顯，整個過程很容易被描述成無縫而有效率。

這也是自動化敘事最容易誤導人的地方。看起來像是機器接手了麻煩工作，實際上常常只是把麻煩拆小、拆散，再交給別人默默吸收。使用者多按幾下、多填幾格、多自己排錯，前線員工多做幾輪確認、多處理系統沒接住的例外，這些都還是勞動，只是不再被明白算進成本裡。

所以談自動化，不能只看多少人力被省下，也要看哪些工作被改名、被轉手、被藏起來。很多所謂順暢，只是因為真正辛苦的部分不再被放在台面上。

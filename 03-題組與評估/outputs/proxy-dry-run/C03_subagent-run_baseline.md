# 第一輪正式結果（subagent-run）

這份結果納入第一輪正式測試紀錄。writer 來源是 subagent，目的在於建立可比較的正式首輪樣本；它不是 OpenAI 或 Claude API 直出結果。

## 正文

政策通常先靠自己宣稱的目標來接受評價，像是效率、透明、簡化程序等等；但真正決定政策長相的，往往不是它寫了什麼，而是它暗中要求執行者額外承擔什麼。一份表格在紙面上看來可以非常簡單，欄位少、步驟清楚，好像誰來都能照著做。實際進入現場後，事情常常完全不是那樣。填表的人可能需要一套沒有明寫的經驗判斷，需要補上制度沒寫全的細節，還得在資訊缺漏時自己想辦法繞路。這些額外勞動很多是無償的，也不會進到正式說明裡。於是政策的表面設計和實際執行之間，就出現一段很深的落差。只看文件，很容易高估制度的清楚程度；回到現場，才會發現它其實是靠大量默會知識與臨場補位撐起來的。
